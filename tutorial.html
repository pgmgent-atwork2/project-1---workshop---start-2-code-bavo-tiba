Memory Game

Hier staat onze code van "Memory Game" zijn

WELCOME TO OUR GAME MEMORY

We gaan een Memory maken!

Daarvoor hebben we drie codetalen nodig

  1. HTML
  2. CSS
  3. JavaScript

Geen paniek, we houden de code eenvoudig, maar maken wel iets superinteractief!

STAP 1 - MAPPEN EN BESTANDEN

  1. Maak een nieuwe map aan (vb op je bureaublad) en geef als naam(Memory)
  2. Open het programma Visual Studio Code. en open de folder die je zopas aangemaakt hebt.
  3. Bouw onderstaande mappen- en bestandenstructuur aan.
  4. Start de live server door rechtsonder de blauwe balk te klikken op Go Live. Jouw standaard browser zal nu gestart worden en je zal een blanco webpagina zien!

STAP 2 - HTML & CSS

Tijd om te coderen, zodat we meer te zien krijgen dan een witte webpagina!

  1. Kopieer onderstaande code in jouw index.html-bestand. Bewaar vervolgens dat bestand.
  2. Kopieer onderstaande code in jouw styles.css-bestand. Bewaar vervolgens dat bestand.

    Dit bevat alvast een aantal basisstijlen die we later zullen nodig hebben voor de look & feel van onze app.

  3. Bekijk het resultaat in de browser. Als alles goed is, ziet het er nu zo uit. Een volledig gekleurd vlak.

STAP 3 - INTERACTIVITEIT MET JAVASCRIPT

3.1 kaart tonen

We halen alle kaart elementen op met querySelectorAll en voegen wat variabelen toe die we later gaan nodig hebben. ook voegen we met een forEach aan elke kaart een EventListener die kijkt of er geklikt wordt op de kaart.

Wanneer er op de kaart geklikt wordt moet er een show class toegevoegd worden aan die kaart. we kijken ook of de kaart onder de juiste omstandigheden is aangeklikt

  • wanneer er 2 kaarten zijn geklikt gaan we niet verder
  • wanneer de kaart al gevonden is gaan we niet verder
  • wanneer de kaart al open ligt gaan we niet verder
als dat allemaal gebeurd is slaan we de kaart op in zijn variable en kijken we of het de eerste of tweede aangeklikte kaart is, wanneer het de tweede is gaan we kijken of de kaarten gelijk zijn.

3.2 kaart controleren

Hier controleren we de kleuren van de kaart die opgeslaan is in de dataset van het element als de kaarten juist zijn gaan we die vastzetten als ze niet juist zijn draaien we ze terug om.

3.3 kaart vastzetten

voor het vastzetten van de kaarten gebruiken we een nieuwe class die "done" noemt hierdoor zal die altijd zichtbaar blijven.

3.4 kaart resetten

Het restten van de kaarten is met een delay zodat de speler tijd heeft om de foute kaarten te bekijken. het bord wordt eerst vastgezet zodat de speler niet meer dan 2 kaarten kan omdraaien. na even te wachten zal de class "show" verwijderd worden zodat de kaarten terug verdwijnen, zal het bord terug klikbaar worden en worden de variable van de kaarten terug op null gezet.

3.5 kaarten random plaats geven

We werken met 16 kaarten in een flex wrap die geen order hebben dus als we met Math.random een random getal geven tussen 0 en 16 zullen de kaarten bij het herladen elke keer een andere plaats hebben. kaarten zullen soms hetzelfde nummer krijgen maar maakt voor de css niet zoveel uit en zal die dan naast elkaar zetten.
